# Release Evidence Tests

This directory contains tests that validate release evidence bundles and promotion eligibility.

## Purpose

These tests are separate from source unit tests because they:
- Validate artifact bundles generated by benchmark runs
- Check promotion policy compliance
- Verify release gate requirements
- Require evidence artifacts from actual benchmark runs

## Test Categories

- **Bundle validation**: Tests that validate the structure and content of evidence bundles
- **Promotion policy**: Tests that verify promotion eligibility checks
- **Release gate**: Tests that validate release gate requirements
- **Artifact integrity**: Tests that verify artifact signatures and provenance

## Running These Tests

```bash
# Run only release evidence tests
pytest tests/release_evidence -v

# Run with specific evidence bundle
pytest tests/release_evidence --evidence-bundle artifacts/bench/shootout/full_logit/results.json
```

## Difference from Source Tests

Source tests (in `tests/`, `rfsn_v10/*/tests/`, `rfsn_v11/tests/`) validate:
- Code correctness
- Algorithm behavior
- API contracts
- Internal invariants

Release evidence tests (in `tests/release_evidence/`) validate:
- Artifact bundle structure
- Promotion policy compliance
- Release gate requirements
- Evidence completeness

## Fix #11

This separation addresses the audit finding that source tests and release-evidence tests were mixed together, making it unclear which tests validate code vs. which tests validate release artifacts.
